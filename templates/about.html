Konsep Mesin Pencarian

Mesin pencarian adalah website yang mengumpulkan dan mengorganisir konten dari seluruh bagian dari internet. Mesin pencarian merupakan cara untuk temu balik informasi. Temu-balik informasi (information retrieval) adalah menemukan kembali (retrieval) informasi yang relevan terhadap kebutuhan pengguna dari suatu kumpulan informasi secara otomatis. Mesin pencarian ini memodelkan query dan term sebagai model ruang vektor. Dan kemudian pengurutan berdasarkan tingkat kemiripannya dengan query.

Pada website ini, pengguna bisa mencari kemiripan dari file-file berekstensi .txt dan kemudian memasukkan kata sebagai querynya. Term didapat dari setiap kata unik dari query.

    Langkah untuk memodelkan query dan term menjadi vektor sebagai berikut :
  1. Lakukan stemming setiap kata pada query ataupun textfile
  2. Lakukan pendataan kata unik untuk seluruh kata (baik query maupun file)
  3. Buatlah sebagai tabel frekuensi kemunculannya untuk masing-masing query maupun file, selanjutnya akan disebut sebagai similiarity table

Pada contoh, query dari user adalah blue cake dan file dari 1.txt 2.txt dan 3.txt hanya berisi 2 kata tersebut. Didapatkan similiarity table seperti diatas yang dapat diperlakukan sebagai vektor.

    Pada gambar representasi vektor-vektornya sebagai berikut :
  • Query : (1i + 1j)
  • 1.txt : (2i + 3j)
  • 2.txt : (1i + 2j)
  • 3.txt : (0i + 1j)

kemudian dilakukan perankingan menggunakan cosine similiarity.

Untuk melihat similiarity dari setiap file .txt, gunakan rumus cosine similiarity dengan A sebagai Query dan B sebagai file yang dicari similiarity-nya.

Cara Menggunakan

Upload file

Pengguna memasukkan file yang akan dicari similiaritynya. File berekstensi .txt

Type query

Pengguna memasukkan file yang akan dicari similiaritynya. File berekstensi .txt

Reset file

Pengguna memasukkan file yang akan dicari similiaritynya. File berekstensi .txt
